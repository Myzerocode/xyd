# Custom Theme
:::subtitle
Learn how to create a custom theme to match your brand and create a unique documentation experience.
:::

# Introduction

To enable a custom theme, create a `.xyd/theme/index.ts` file in your project:

```
.
├─ docs                # project root
│  ├─ .xyd
│  │  └─ theme
│  │     └─ index.ts   # theme entry point
│  │     └─ index.css  # custom styles (optional)
│  │  
│  └─ index.md
└─ xyd.json            # settings file
```

<code>xyd</code> will automatically use your custom theme when it detects the presence of a theme entry file. You have two options for customization:

1. Create a completely custom theme from scratch
2. Extend the default theme to make targeted customizations while maintaining the core design

## Theme Structure

The theme system consists of two main files:

- `index.ts`: The theme entry point where you define your theme configuration
- `index.css (optional)`: Custom CSS styles for additional styling needs

## Theme Interface

The theme entry file must export a default function that receives the following parameters:

```ts
// You can directly import Vue files in the theme entry
// VitePress is pre-configured with @vitejs/plugin-vue.
import Layout from './Layout.vue'

export default {
  Layout,
  enhanceApp({ app, router, siteData }) {
    // ...
  }
}
```
